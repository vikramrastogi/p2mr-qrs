# P2MR SLH-DSA Leaf

Draft-stage pre-review package for one narrow post-quantum signing primitive in
the P2MR / BIP-360 discussion.

This is not activation-ready consensus evidence.

Start here: docs/review/start-here.md

## What This Is

This repository proposes and tests one P2MR-only future leaf version:

- one SLH-DSA-SHA2-128s public key;
- one SLH-DSA-SHA2-128s signature;
- SIGHASH_DEFAULT only;
- no legacy-output freeze, burn, throttle, sunset, or rescue policy;
- no witness discount;
- no algorithm registry;
- no tapscript opcode in v1.

The design is intended to complement BIP-360 by making one concrete SLH-DSA
future-leaf rule reviewable. It does not replace BIP-360 and does not ask for
activation support.

## Publication Gate

Publication or pre-review release readiness is defined only by:

```sh
bash scripts/release_check.sh
```

A quick benchmark, vector check, or generated report outside the release gate is
advisory unless regenerated by that gate from the current source tree. Checked-in
`out/` files remain sample artifacts.

For stricter standard-run evidence:

```sh
bash scripts/full_release_check.sh
```

Both gates require OpenSSL 3.5 or newer with a provider exposing
`SLH-DSA-SHA2-128s` for strict QRS verification checks.

## Current Status

- Native OpenSSL SLH-DSA-SHA2-128s timing is measured when the local OpenSSL
  build exposes the algorithm through EVP.
- Native libsecp256k1 individual BIP-340 Schnorr timing is measured.
- Experimental native BIP-340 batch timing is sensitivity analysis only.
- Reviewed public libsecp256k1 batch verification remains explicitly
  unavailable in this package.
- The QRS validation path is a local model, not Bitcoin Core consensus
  integration.
- Current vectors are provisional executable fixtures and draft cryptographic
  vectors, not final serialized consensus vectors.

Known blockers before advancing beyond Draft are tracked in
docs/evidence/consensus-gap-manifest.md.
